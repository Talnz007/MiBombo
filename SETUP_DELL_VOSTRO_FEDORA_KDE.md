# Dell Vostro 13th Gen (i7, DDR5) — Fedora KDE Setup & Deployment Guide

This guide details how to set up the Automated OSINT Operations Platform on a Dell Vostro PC equipped with an Intel Core i7 13th Gen processor and DDR5 memory, running the Fedora KDE operating system.

## 1. System Preparation & Dependencies

Since you are running Fedora KDE, you will use `dnf` for package management. Your 13th Gen i7 and fast DDR5 memory will provide excellent performance for the local LLM processing (Ollama) and concurrent scraping.

1. **Update your system:**
   ```bash
   sudo dnf update -y
   ```

2. **Install prerequisite packages:**
   ```bash
   sudo dnf install -y python3 python3-pip python3-virtualenv git gcc python3-devel cronie
   ```

3. **Enable and start the Cron service:**
   Fedora might not have the cron service running by default.
   ```bash
   sudo systemctl enable crond.service
   sudo systemctl start crond.service
   ```

## 2. Setting Up the Repository and Environment

1. **Clone or Navigate to the Repository:**
   If you are moving your existing `/home/talnz/automatingwork` directory:
   ```bash
   cd /home/talnz/automatingwork
   ```
   *If starting fresh, `git clone` your repository first.*

2. **Create and Activate a Virtual Environment:**
   ```bash
   python3 -m venv venv
   source venv/bin/activate
   ```

3. **Install Python Requirements:**
   ```bash
   pip install --upgrade pip
   pip install -r requirements.txt
   pip install -r osint_news_scraper/requirements.txt
   ```

4. **Install Playwright Browsers:**
   Playwright requires specific browser binaries.
   ```bash
   playwright install
   playwright install-deps
   ```
   *(Note: `install-deps` on Fedora will install necessary browser dependencies via `dnf`.)*

## 3. Setting Up Ollama (Local AI)

Your Dell Vostro's i7 13th Gen CPU and DDR5 memory will handle Ollama well, even without a dedicated GPU.

1. **Install Ollama:**
   ```bash
   curl -fsSL https://ollama.com/install.sh | sh
   ```

2. **Start the Ollama Service:**
   ```bash
   sudo systemctl enable ollama
   sudo systemctl start ollama
   ```

3. **Pull Required Models:**
   The pipeline utilizes these models for local processing.
   ```bash
   ollama run dolphin-llama3
   ollama run qwen3-vl:4b
   ```
   *(Press `Ctrl+D` to exit the Ollama prompt after the models finish downloading).*

## 4. Managing Cookies

The pipeline requires Twitter cookies in `.pkl` format to bypass rate limits. 

1. Ensure your cookies are placed in the `cookies/` directory as `.pkl` files. 
2. If you obtain new `.json` cookies, you can convert them using the provided scripts. 
3. The system will automatically rotate through the available `.pkl` files.
4. **No Duplicates:** Ensure you don't have multiple `.pkl` files belonging to the exact same Twitter account to prevent redundant logins or overlapping sessions. The script automatically rotates them.

## 5. Running the Pipeline

1. **Activate the Virtual Environment:**
   ```bash
   cd /home/talnz/automatingwork
   source venv/bin/activate
   ```

2. **Run the Scrapers:**
   You can run the global or regional pipelines. It is recommended to run them in separate terminal tabs or use `tmux`/`screen`/`Konsole` tabs.
   ```bash
   # Terminal 1: Global Pipeline
   python osint_news_scraper/main.py

   # Terminal 2: Regional Pipeline
   python regional_news/main.py
   ```

3. **First-Time WhatsApp Authentication:**
   On the first run, Playwright will open a Chromium window for WhatsApp Web. You must scan the QR code using your phone. The session will be saved in the `whatsapp_session/` directory for future use.

## 6. Automating with Cron (Optional)

To automatically sync the JSON reports generated by the dashboard pipeline:

1. **Open crontab:**
   ```bash
   crontab -e
   ```

2. **Add the sync job:**
   ```bash
   0 * * * * /home/talnz/automatingwork/sync_reports.sh >> /home/talnz/automatingwork/sync.log 2>&1
   ```
